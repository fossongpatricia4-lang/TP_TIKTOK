1. Comprendre ce qu'on va construire
On va creer un clone de TikTok avec React Native CLI. C'est une application mobile multiplateforme (iOS
& Android) avec une interface graphique complete. L'utilisateur voit les videos sur son telephone, peut les
uploader, liker, commenter et gerer son profil.
Ce que l'application React Native va pouvoir faire :
• S'inscrire et se connecter (Firebase Authentication)
• Voir un fil de videos en defilement vertical (comme TikTok)
• Uploader une video depuis la galerie ou la camera
• Liker et commenter une video
• Voir et modifier son profil utilisateur
• Suivre d'autres utilisateurs
• Notifications push (Firebase Cloud Messaging)

2. L'architecture globale
Application React Native CLI | +-- Firebase Authentication -> gere la
connexion/inscription +-- Cloud Firestore -> stocke users, videos, likes, commentaires
+-- Firebase Storage -> stocke les fichiers video +-- Firebase Cloud Messaging ->
notifications push +-- React Navigation -> navigation entre les ecrans

ETAPE 3 — Chaque personne code son ecran
DEV 2 — Authentification (LoginScreen.tsx & RegisterScreen.tsx)
Ce que tu dois faire :
• Creer un formulaire avec TextInput pour email + mot de passe
• Utiliser createUserWithEmailAndPassword (Firebase Auth) pour l'inscription
• Utiliser signInWithEmailAndPassword pour la connexion
• Apres connexion, naviguer vers l'ecran Home avec navigation.replace('Home')
• Sauvegarder l'utilisateur dans Firestore collection 'users' apres inscription
• Afficher les erreurs Firebase avec un message lisible

DEV 3 — Fil de videos (HomeScreen.tsx & VideoCard.tsx)
Ce que tu dois faire :
• Recuperer les videos depuis Firestore (getDocs sur collection 'videos')
• Afficher les videos avec un FlatList vertical (paginated)
• Chaque VideoCard prend TOUT l'ecran (height: '100%', like TikTok)
• Utiliser expo-av pour lire la video automatiquement quand elle est visible
• Mettre en pause les videos non visibles (onViewableItemsChanged)
• Afficher : description, username, bouton like, bouton commentaire

DEV 4 — Upload de video (UploadScreen.tsx)
Ce que tu dois faire :
• Utiliser expo-image-picker pour choisir une video depuis la galerie
• Demander la permission d'acces a la galerie avec MediaLibrary.requestPermissionsAsync()
• Uploader la video sur Firebase Storage avec uploadBytes()
• Recuperer l'URL de telechargement avec getDownloadURL()
• Sauvegarder dans Firestore : { userId, videoUrl, description, likesCount: 0, createdAt }
• Afficher une barre de progression pendant l'upload

DEV 5 — Likes & Commentaires (LikeButton.tsx & CommentsScreen.tsx)
Ce que tu dois faire :
• Likes : verifier si l'utilisateur a deja like (chercher dans collection 'likes')
• Si pas like : addDoc dans 'likes' + incrementer likesCount dans 'videos'
• Si deja like : deleteDoc dans 'likes' + decrementer likesCount
• Changer la couleur du coeur selon l'etat (rouge = like, gris = pas like)
• Commentaires : afficher les commentaires avec FlatList, permettre d'en ajouter
• Sauvegarder le commentaire dans Firestore et rafraichir la liste

DEV 6 — Profil (ProfileScreen.tsx)

Ce que tu dois faire :
• Afficher la photo de profil, le nom, la bio, nombre de followers/following
• Afficher toutes les videos de l'utilisateur dans une grille 3 colonnes
• Permettre de modifier le nom et la bio avec TextInput
• Permettre de changer la photo de profil avec expo-image-picker
• Sauvegarder les modifications avec updateDoc dans Firestore
• Distinguer le profil de l'utilisateur connecte et les autres profils

DEV 7 — Navigation (navigation/AppNavigator.tsx)
Ce que tu dois faire :
• Installer et configurer NavigationContainer dans App.tsx
• Creer un Stack Navigator pour Auth (Login, Register)
• Creer un Bottom Tab Navigator pour l'app principale (Home, Upload, Profile)
• Gerer l'etat d'authentification : si connecte -> Tabs, sinon -> Stack Auth
• Utiliser onAuthStateChanged (Firebase) pour detecter automatiquement la connexion
• Ajouter des icones sur chaque onglet avec @expo/vector-icons

DEV 8 — UI/Style (utils/theme.ts)
Ce que tu dois faire :
• Creer un fichier theme.ts avec les couleurs globales : noir, blanc, rouge TikTok (#FF0050)
• Creer des styles communs reutilisables (boutons, textes, cartes)
• S'assurer que tous les ecrans ont le meme look and feel
• Ajouter des icones avec @expo/vector-icons (Ionicons recommande)
• Gerer les safe areas (notch iOS, barre Android) avec react-native-safe-area-context
• S'assurer que l'app est responsive sur differentes tailles d'ecran
